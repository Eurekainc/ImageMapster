Release Notes

This section details major changes to each release. For more detail see the commit history on github.

Version 1.2.5

Features:

  • Zepto compatible: ImageMapster will work with Zepto instead of jQuery using a special build to patche Zepto with a couple jQuery features that are not implemented by Zepto. This build is included in the distribution. The Zepto build is completely compatible with jQuery as well and will detect when jQuery is being used. But because it adds a little bit to the size of the plugin I'm keeping it as a separate build.
  • Allow passing options to set to control rendering options at run-time.
  • keys option allows accessing keys for an area, area group or key group.
  • mouseoutDelay option allows a time delay after the mouse exits a hotspot before the highlight is removed.
  • highlight method allows control of highlighted area from code. This is different from the existing set method, which selects or deselect areas. The new feature mimics the effect of a mouseover, versus a click
  • Returning true from an onClick event handler will cause navigation to the area's href url

Bug fixes and improvements:

  • Rendering options can be changed on the fly without affecting areas that have already been rendered. Previously, while you could use set_options to change any rendering option to a map that had already been bound, deselecting areas would cause everything to be redrawn with the new options.
  • Enable fade effect for IE 8 (requires applying opacity to child elements, IE8 specific problem)
  • Improve rendering sharpness for 1 pixel strokes by adding 0.5 pixels to coordinates that fall on a pixel boundary
  • Block UI events while resizing, would cause unpredictable behavior
  • Detect touchscreen devices and ignore mouseenter events improves mobile device performance
  • Fix flickering tooltips and areas when moving between adjacent grouped areas
  • Substantial performance improvements with large image maps
  • (bug) Fix tooltip positioning for circle and rect area shapes
  • (bug) Trim results of string splits so spaces don't cause problems. Spaces between keys in a list would cause failure.
  • (bug) Couldn't disable fill effect when using altImage
  • (bug) Fix for mask handling in Firefox 6.0

General:

  • Method queuing: All method calls are queued, and will be executed only after any pending async operations are finished (initial binding or resizing). Because ImageMapster binds asynchronously (as it needs to wait for images to finish loading), and the resize method can be called with effects that take time, this was not safe prior to 1.2.5.
  • Refactor code into a modular architecture
  • Split into two separate builds (one for jQuery only, one for jQuery or Zepto)
  • Works if Explorer Canvas (a library to make HTML5 code work with Internet Explorer) is loaded. Though Excanvas isn't needed for ImageMapster, if it was included on a page, ImageMapster would fail.
